Чим інтерфейс Set відрізняється від Collection та List?

Set: не допускає дублікатів, порядок елементів не гарантується.
Collection: базовий інтерфейс для роботи з колекціями, включає Set і List.
List: допускає дублікатів, зберігає порядок елементів.
HashSet відрізняється від TreeSet?

HashSet: базується на хеш-таблиці, не гарантує порядку елементів, швидкий доступ O(1).
TreeSet: базується на дереві (Red-Black Tree), зберігає елементи у відсортованому порядку, доступ O(log n).
Що таке хешування, хеш, хеш-код, хеш-функція?

Хешування: процес перетворення об'єкта у числове значення (хеш-код) для швидкого пошуку.
Хеш: результат роботи хеш-функції.
Хеш-код: числове значення, яке повертає метод hashCode().
Хеш-функція: алгоритм, що обчислює хеш-код.
Вимоги до коректної хеш-функції:

Постійність: однаковий об'єкт завжди повертає однаковий хеш-код.
Взаємозв'язок із методом equals: якщо два об'єкти рівні (equals() повертає true), їх хеш-коди мають бути однаковими.
Властивості гарної хеш-функції:

Мінімізація колізій (різні об'єкти мають різні хеш-коди).
Швидкість обчислення.
Рівномірний розподіл хеш-кодів.
Критерії для об'єктів у HashSet:

Реалізація методів hashCode() і equals() з урахуванням логічної рівності.
Переваги/недоліки HashSet порівняно з TreeSet:

Переваги: швидший доступ (O(1)), менші витрати пам'яті.
Недоліки: не гарантує порядку елементів.
TreeSet забезпечує сортування, але повільніший (O(log n)).
Чим size відрізняється від capacity?

size: кількість елементів у колекції.
capacity: кількість доступних слотів для зберігання.
size не може бути більшим за capacity, але capacity може бути більшим за size (резерв для нових елементів).
Що таке load factor та на що він впливає?

load factor: коефіцієнт заповнення (size/capacity), при перевищенні якого хеш-таблиця розширюється.
Впливає на продуктивність та використання пам'яті.
Що таке асоціативний масив? Чим ключ відрізняється від значення?

Асоціативний масив: структура даних, що зберігає пари "ключ-значення".
Ключ: унікальний ідентифікатор для доступу до значення.
Значення: дані, які зберігаються і доступні за ключем.
Чи може бути кілька однакових ключів або значень?

Ключі: не можуть дублюватися.
Значення: можуть дублюватися.
Наслідки зберігання неімутабельних об'єктів у HashSet/HashMap:

Модифікація об'єкта може змінити його хеш-код, що призводить до проблем із пошуком або видаленням.
Чи можуть об'єкти без Comparable зберігатись у TreeMap?

Ні, якщо не вказано окремий Comparator.
Чи можуть об'єкти без Comparable зберігатись у HashMap?

Так, наявність Comparable не впливає на роботу HashMap.
